## 背景

CEO 对 label generation pipeline 进行了全面审查和修正, 涉及 5 项重大改动:

### 改动清单

1. **BUG-19 v3 — z-convention fix**: `_get_corners_lidar` 中 z 是 box BOTTOM (mmdet3d convention), 不是 center. 修复前所有车辆只有下半部分被覆盖.

2. **Convex hull 替代 AABB**: 保留 convex hull (`use_rotated_polygon=True`), 比 AABB 紧 ~25%. AABB 对斜角车辆严重过大.

3. **IoF/IoB 对 hull 多边形计算 (Sutherland-Hodgman)**: 之前 IoF/IoB 对 AABB 计算, convex hull 内部的 cell IoF 永远 ~1.0 导致过滤失效. 现在用 Sutherland-Hodgman 算法精确计算 cell 与 hull 多边形的交集面积.

4. **filter_invisible=False**: 关闭 nuScenes vis_token 过滤 (vis_token==1 即 0-40% 可见). 我们有自己的几何 vis_ratio 过滤, 两者重复且 nuScenes 的过滤过于激进.

5. **vis + cell_count 组合过滤**: 纯 vis_ratio < 10% 会误杀画面中占大面积的半出画车辆 (如 vis=7.3% 但占 29 个 cell). 新逻辑: **仅当 vis < 10% AND hull_cells < 6 时才过滤**.

### 最终 config 参数
```python
filter_invisible=False,
use_rotated_polygon=True,
min_vis_ratio=0.10,
min_vis_cell_count=6,    # vis 低但占 >=6 cell 的大目标保留
min_iof=0.30,            # IoF/IoB 对 hull polygon 计算
min_iob=0.10,
grid_assign_mode='overlap',
```
