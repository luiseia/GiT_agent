## 背景

`/home/UNT/yz0370/projects/GiT/` 主目录空间不足。
`/home/UNT/yz0370/projects/GiT/ssd_workspace` 是软链接到大容量 SSD 盘。
需要将 work_dirs 迁移到 SSD 并建立软链接, 保证后续训练和现有路径不受影响。

## 任务

将以下两个目录迁移到 SSD:

### 1. 迁移 work_dirs
```bash
# 1. 移动到 SSD
mv /home/UNT/yz0370/projects/GiT/work_dirs /home/UNT/yz0370/projects/GiT/ssd_workspace/work_dirs

# 2. 创建软链接 (保证现有路径兼容)
ln -s /home/UNT/yz0370/projects/GiT/ssd_workspace/work_dirs /home/UNT/yz0370/projects/GiT/work_dirs
```

### 2. 迁移 work_dirs_12
```bash
mv /home/UNT/yz0370/projects/GiT/work_dirs_12 /home/UNT/yz0370/projects/GiT/ssd_workspace/work_dirs_12

ln -s /home/UNT/yz0370/projects/GiT/ssd_workspace/work_dirs_12 /home/UNT/yz0370/projects/GiT/work_dirs_12
```

## 约束

- **迁移前确认**: 没有正在写入这些目录的进程 (P5 已完成, GPU 已释放)
- **迁移后验证**: `ls -la` 确认软链接正确, `ls` 确认内容可访问
- **检查 ssd_workspace 剩余空间**: `df -h` 确认 SSD 有足够空间
- **不要删除任何文件**, 只做移动 + 软链接

## 验收标准

- [ ] `work_dirs` 和 `work_dirs_12` 物理位置在 `ssd_workspace/` 下
- [ ] 原路径 `/home/UNT/yz0370/projects/GiT/work_dirs` 和 `work_dirs_12` 是有效软链接
