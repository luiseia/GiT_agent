## 三项修复 (全部必须)

### 修复 1: LR Milestones 修正 (BUG-17)

**问题**: MMEngine MultiStepLR milestones 为相对于 begin 的值。P5 config milestones=[4000,5500] + begin=1000, 实际 decay @5000, 二次 decay @6500 永不触发。

**修复**:
```python
# 新 config:
warmup_iters = 500      # 缩短 warmup (Critic 建议)
begin = 500              # 对应 warmup 结束
milestones = [2000, 3500]  # 相对 begin=500, 实际 decay @2500, @4000
max_iters = 6000         # 保持不变
```

**验证**: 训练开始后检查 log, 确认 lr 在 iter 2500 发生变化。

### 修复 2: per_class_balance → sqrt 加权 (BUG-17)

**问题**: 等权 per_class_balance 让极少类 (trailer: 90 GT vs car: 8269 GT, 92:1) 的梯度噪声主导 batch, 导致类别零和振荡。

**修复**: 在 `generate_occ_flow_labels.py` 的权重计算中:
```python
# 旧: 等权 (每个类别的 GT 权重 = 1/count_c)
# 新: sqrt 加权
weight_c = 1.0 / math.sqrt(count_c / min_count)
# 效果:
#   car (8269):   weight = 1/sqrt(91.9) ≈ 0.104
#   truck (359):  weight = 1/sqrt(3.99) ≈ 0.500
#   bus (188):    weight = 1/sqrt(2.09) ≈ 0.691
#   trailer (90): weight = 1/sqrt(1.00) = 1.000
# 对比等权: 全部 = 1.0 → 差异从 92:1 缩小到 9.6:1
```

**代码位置**: `generate_occ_flow_labels.py` 中 `per_class_balance` 相关权重计算。
**约束**: 用 config 参数 `balance_mode='sqrt'` 控制, 保持 `balance_mode='equal'` 的旧行为可用。

### 修复 3: 双层投影替换 Linear 压缩 (CEO 决策)

**问题**: Linear(4096,768) 的 5.3:1 压缩迫使不同类别共享子空间, 改善一类时破坏另一类。

**修复**: 在 `preextracted_feature_embed.py` 中:
```python
# 旧:
self.proj = nn.Linear(4096, 768)

# 新:
self.proj = nn.Sequential(
    nn.Linear(4096, 1024),
    nn.GELU(),
    nn.Linear(1024, 768)
)
# 压缩比: 4096→1024 (4:1) → 768 (1.33:1), 中间层保留更多信息
```

**代码位置**: `mmdet/models/backbones/preextracted_feature_embed.py`
**约束**: 用 config 参数控制 (如 `proj_hidden_dim=1024`), 默认使用新方案。
