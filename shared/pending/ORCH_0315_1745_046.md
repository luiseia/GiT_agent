## ⭐ 修改 5: BEV 空间增强 (代码实现)

**参考**: 3D Detection 标准做法 (PointPillars/SECOND nuScenes config)

### 5a. RandomFlip3D (水平翻转, prob=0.5) — 必做

**原理**: 沿 BEV x 轴水平翻转，打破"固定位置总有目标"的空间先验。

**需要翻转的内容**:
1. **相机图像**: 水平翻转每张图像
2. **3D annotations** (在 `LoadAnnotations3D_E2E` 之后、`GenerateOccFlowLabels` 之前):
   - box center x: `x → -x` (BEV 坐标系)
   - heading/rotation: `yaw → -yaw` 或 `yaw → π - yaw` (取决于坐标系约定)
   - 其他属性 (y, z, w, l, h) 不变
3. **相机外参** (如果 grid_interpolate 用到): 对应调整

**实现位置**: 在 train_pipeline 中，`LoadFrontCameraImageFromFile` 之后、`GenerateOccFlowLabels` (即 pipeline_common) 之前插入

```python
train_pipeline = [
    dict(type='LoadAnnotations3D_E2E', ...),
    dict(type='LoadFrontCameraImageFromFile', ...),
    dict(type='PhotoMetricDistortion'),
    # ★ 新增: BEV 水平翻转
    dict(type='RandomFlipBEV', flip_ratio=0.5),
    dict(type='ResizeForOccInput', target_size=(1120, 1120)),
    dict(**pipeline_common, filter_invisible=False),
    ...
]
```

**实现要点**:
- 检查 `LoadAnnotations3D_E2E` 输出的 annotation 格式（`gt_bboxes_3d`, `gt_labels_3d` 的结构）
- 确认 BEV 坐标系约定：x 正方向是前还是右？
- 翻转后需确保 `GenerateOccFlowLabels` 生成的 grid targets 正确

## 修改 6: BUG-45 — 推理时添加 causal attention mask

**问题**: 训练时 decoder 有 causal mask (`git.py:525` → `get_attn_mask()`)，推理时 `git_occ_head.py:1126` 硬编码 `attn_mask=None`。

**训练 vs 推理不一致**:
| 阶段 | attn_mask | 约束 |
|------|-----------|------|
| 训练 | `get_attn_mask()` 生成 causal mask | 位置 i 只能看 [0, i-1] |
| 推理 | `None` | 无约束 |

**修复**: 在 `git_occ_head.py` 的 `decoder_inference()` 中，为每个 `pos_id` 步骤生成对应的 causal mask，与训练时一致。

**修复位置**: `git_occ_head.py:1124-1127`
```python
# 当前代码:
curr_x, pre_kv_update = layer.token_forward(
    ..., attn_mask=None, ...)

# 修复后: 生成与训练一致的 causal mask
# 可从 git.py 的 get_attn_mask() 提取逻辑，或直接引用
curr_x, pre_kv_update = layer.token_forward(
    ..., attn_mask=causal_mask, ...)
```

**注意**: 需要确认 `pre_kv` 缓存机制下 mask 的形状是否需要调整（只有当前 token 作为 query，pre_kv 作为 key/value 时，mask 维度可能不同于训练时的全序列 mask）。
