## Task 2: Plan P2 — 2048+GELU 纯 BUG-39 修复 (~2h)

### 目的

Plan P 失败因超参错误 (lr_mult=1.0, warmup=100)。Plan P2 使用 P6 完全相同的超参, **仅加 GELU**。这是唯一能干净分离 GELU 变量的实验。

### Config

从 `plan_p6_wide_proj.py` 复制为 `plan_p2_wide_gelu_fix.py`, **仅改一处**:

```python
# ★★ 唯一改动:
preextracted_proj_use_activation = True    # 加 GELU! (P6 是 False = BUG-39)

# 以下全部保持与 P6 一致:
preextracted_proj_hidden_dim = 2048        # 与 P6 一致
# lr_mult = 2.0                            # 与 P6 一致 (不要改成 1.0!)
# warmup = 500                             # 与 P6 一致 (不要缩短!)
# milestones = [2000, 4000]                # 与 P6 一致
# bg_balance_weight = 2.5                  # 与 P6 一致
```

**关键注意**:
1. **lr_mult 必须是 2.0** (与 P6 一致, 不是 Plan P 的 1.0)
2. **warmup 必须是 500** (与 P6 一致, 不是 Plan P 的 100)
3. **milestones 保持 [2000, 4000]** (与 P6 一致)
4. **max_iters = 2000** (比 Plan L 长, 足够收敛; 比 P6 的 6000 短, 快速验证)
5. **val_interval = 500** (获取 @500, @1000, @1500, @2000 共 4 个 ckpt)
6. **load_from = P5b@3000** (与 P6 相同起点)
7. **单 GPU 训练** (所有指标可信, 不需后续 re-eval)
