## Task 1: 实现 OnlineDINOv3Embed 多层特征支持

修改 `GiT/mmdet/models/backbones/vit_git.py` 中的 `OnlineDINOv3Embed` 类:

**目标**: 支持从多层提取特征并拼接，向后兼容单层模式。

**关键改动**:

1. `__init__` 新增参数 `online_dinov3_layer_indices` (list, 默认 None):
   - 如果提供 (如 `[10,20,30,40]`)，从多层提取并拼接
   - 如果为 None，走原来的 `online_dinov3_layer_idx` 单层路径（向后兼容）
   - 当使用多层时，投影层输入维度自动变为 `len(layer_indices) * embed_dim`

2. `forward` 中:
   ```python
   # 多层模式
   if self.layer_indices is not None:
       features = self.dinov3_model.get_intermediate_layers(
           img_tensor, n=self.layer_indices, reshape=False, norm=True
       )
       # 每层: (B, N_patches, 4096) → concat: (B, N_patches, 16384)
       combined = torch.cat(features, dim=-1)
   # 单层模式 (向后兼容)
   else:
       features = self.dinov3_model.get_intermediate_layers(...)
       combined = features[0]
   
   # 投影
   projected = self.proj(combined)  # (B, N_patches, 768)
   ```

3. 投影层适配:
   - 多层模式: `nn.Linear(16384, 2048)` + GELU + `nn.Linear(2048, 768)`
   - 单层模式: 保持不变 `nn.Linear(4096, 2048)` + GELU + `nn.Linear(2048, 768)`

4. **可选**: 在拼接后、投影前加 `nn.BatchNorm1d`（论文标准做法）

**重要约束**:
- 必须向后兼容: `online_dinov3_layer_indices=None` 时行为与当前完全一致
- ORCH_029 的 config 不受影响
- 不要改 DINOv3 模型本身的代码
