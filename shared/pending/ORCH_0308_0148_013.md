## 问题分析

### 代码位置

**训练代码** (`generate_occ_flow_labels.py` L536-539):
```python
box_center_geo = box_params.copy()
box_center_geo[2] += box_params[5] * 0.5  # z += h/2, 将 z 从底部移到中心?
corners_lidar = self._get_corners_lidar(box_center_geo[:7])
```

**可视化代码** (`visualize_polygon_vs_aabb.py` L259-261):
```python
box_geo = box.copy()
box_geo[2] += box[5] * 0.5  # 同样的 z += h/2 调整
```

### 问题

Conductor 对比了 WITH 和 WITHOUT `z += h/2` 的投影结果：
- **WITH z+=h/2**: 投影多边形只覆盖车辆上半部分，底部/车轮区域被截断
- **WITHOUT z+=h/2**: 投影中心下移 30-65 pixels，更贴近车辆在图像中的实际位置
- 角点 z_lidar 范围对比 (例: car): WITH [0.00, 1.96] vs WITHOUT [-0.98, 0.98]

**nuScenes 标注约定**: z 是 3D box **中心**的高度。如果 z 已经是中心，`z += h/2` 会把它移到 box **顶部**，导致投影覆盖区域偏高。

### 但有疑点需要验证

可能 pkl 中的 z 不是原始 nuScenes 中心高度，而是被预处理降到了底部高度。需要 Admin 验证：
1. 读取 pkl 中几个典型车辆的 z 值和 h 值
2. 对照 lidar 坐标系: lidar 高度 1.84m (from lidar2ego_translation z=1.84)
3. 判断 z 是中心高度还是底部高度

### 2. 根据验证结果修复

**如果 z 是中心高度 (nuScenes 标准)**:
- **移除** `z += h/2`，`_get_corners_lidar` 应该直接从中心生成 8 个角点
- 修改 `generate_occ_flow_labels.py` L537: 删除 `box_center_geo[2] += box_params[5] * 0.5`
- 修改 `visualize_polygon_vs_aabb.py` L260: 删除 `box_geo[2] += box[5] * 0.5`

**如果 z 是底部高度 (预处理后)**:
- `z += h/2` 是正确的（从底部移到中心）
- 问题可能在 `_get_corners_lidar` 的实现——检查它是否假设 z 是中心
