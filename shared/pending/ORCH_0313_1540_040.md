## 背景 — ORCH_036 为何失败

ORCH_036 的 4 个 eval 结果**完全相同**. 原因:
1. `_predict_single()` (git_occ_head.py L1515) 中 `ret_scores = bboxes.new_ones(...)` — 所有 score = 1.0
2. `Occ2DBoxMetric.__init__` (occ_2d_box_eval.py L22) 接收 `score_thr` 参数但**从未使用** (未存储到 self)
3. 无论设什么 score_thr, 结果都一样

## 修复方案: 在 Evaluator 中实现 score_thr 过滤

### 方案选择

**不修改模型 predict 输出** (风险最低), 而是在 Evaluator 的 `process()` 方法中, 基于 `grid_markers` 的 token probability 实现过滤:

1. 在 `Occ2DBoxMetric.__init__` 中保存 `self.score_thr = score_thr`
2. 在 `process()` 方法中, 对每个 grid slot:
   - 如果 marker token 的 softmax confidence < score_thr → 视为 background (跳过该 slot)
   - 这需要模型在 predict 阶段输出 marker token 的 confidence

**备选 (更简单)**: 直接在 evaluator process() 中用 `pred_grid_markers` 的值过滤:
- marker_token == background → 跳过
- 这实际上是 `score_thr` 的简化版本
