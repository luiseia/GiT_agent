> "高度截断是我们获取平面真值box的方式在分配grid正负样本的设计中不应该被使用。这个问题必须马上修复然后重新可视化我要检查"

## 问题描述 (BUG-19)

CEO 从 polygon_viz 结果 (`/mnt/SSD/GiT_Yihao/polygon_viz/`) 中发现：
很多有车的 grid 并没有被分配为正样本。

**根因**: `proj_z0 = -0.5` (用于获取 BEV 平面真值 box 的投影平面高度) 被错误地用于 grid 正负样本分配逻辑。

**CEO 澄清**: `proj_z0` 仅应用于定义 BEV 平面投影高度 (3D→BEV 转换时的 z 平面)。在 grid 正负样本分配设计中，不应使用此参数。

## 修复任务

### 1. 调查并修复 `generate_occ_flow_labels.py`

**关键代码位置**:

- **L421 `_generate_bev_references`**:
  ```python
  zs = np.full_like(xs, self.proj_z0, dtype=np.float32)  # BEV grid 中心点 z=-0.5
  ```
  此函数生成 `valid_mask` (哪些 BEV cell 在相机 FOV 内), 作为 `image_reference_valid_mask` 传给模型。
  检查此 mask 是否影响 grid 正负样本分配或 loss 计算。

- **L523-583 (GT box → image grid 分配循环)**:
  检查 3D box 投影到 image grid 的过程中是否有隐式的高度过滤。
  特别注意 `valid_z = corners_cam[:, 2] > 0` (L548) 和 `keep_front` 过滤 (L576-577)。

- **L598-600 (keep_front 过滤)**:
  `bev_boxes_final = bev_boxes_all[keep_front]` — 只保留投影到相机内的 GT。
  检查这是否导致 BEV 空间内有车但不可见的 cell 丢失。

**修复原则**:
- `proj_z0` 只用于 3D→BEV 平面投影 (获取真值 box)
- grid 正负样本分配应基于 BEV 空间的 box-grid 覆盖关系, 不应依赖 z 高度过滤
- 如果 `image_reference_valid_mask` 不影响训练 (head 中仅在 docstring 提及), 可将其设为全 True
