## 实验 B: 从 iter_100 Resume + 降低 LR

**假设**: 较低的 max_lr 能延缓 grid_pos_embed shortcut 的固化速度，让模型保持更长的健康期

**Config 改动**:
- `load_from` 或 `resume_from`: `/mnt/SSD/GiT_Yihao/Train/Train_20260316/full_nuscenes_large_v1_orch055/iter_100.pth`
- `max_lr`: 当前值的 1/5（如 base_lr 从 5e-5 降到 1e-5）
- 其余所有参数保持 ORCH_055 配置不变

**启动方式**: 必须 2-GPU DDP（BUG-78）

```bash
CUDA_VISIBLE_DEVICES=2,3 python -m torch.distributed.launch \
  --nproc_per_node=2 --master_port=29510 \
  tools/train.py configs/GiT/plan_full_nuscenes_large_v1.py \
  --work-dir /mnt/SSD/GiT_Yihao/Train/Train_20260316/full_nuscenes_large_v1_orch056 \
  --launcher pytorch --resume /mnt/SSD/GiT_Yihao/Train/Train_20260316/full_nuscenes_large_v1_orch055/iter_100.pth
```

**Frozen-check**: @200, @300, @400, @500（不早停，收集完整轨迹）

**期望**: 如果 LR 是模板化速度的关键因素，降低 LR 后:
- @200 的 marker_same 应低于 ORCH_055 @200 的 0.962
- 相变时间点应推迟（不再在 @300→@400）
