## 背景

AABB→旋转多边形修复 (P4 验证) 使用 ConvexHull + 叉积法判断 grid cell 是否在投影多边形内。
当前 `viz_range_positive_targets()` 只可视化了分配结果 (哪些 cell 被选中), 但**看不到旋转多边形本身的形状**。
CEO 需要可视化来确认修复的正确性。

## 目标

在 `generate_occ_flow_labels.py` 的 grid 分配过程中, 增加可视化:
1. 在图像上绘制每个 GT 的**旋转多边形 (凸包) 轮廓** (红色)
2. 在同一图像上绘制原始 **AABB 矩形** (蓝色虚线), 用于对比
3. 被旋转多边形排除的 cell 用黄色标记, 保留的 cell 用绿色标记
4. 保存到 `/home/UNT/yz0370/projects/GiT/ssd_workspace/polygon_viz/` 目录 (主目录空间不足, 必须用 SSD)

## 技术细节

### 关键代码位置
- `generate_occ_flow_labels.py` L544-558: 旋转多边形分配逻辑
  - `hull_pts`: ConvexHull 顶点 (凸包, 图像坐标)
  - `_point_in_convex_hull()`: 叉积判断
  - `_get_covered_cells()`: 计算覆盖的 cell 列表

### 可视化内容
```
对每个 sample 生成一张图:
- 底图: 原始前摄像头图像
- 红色多边形: hull_pts 连线 (旋转多边形/凸包)
- 蓝色虚线矩形: 原始 AABB (u_min, v_min, u_max, v_max)
- 绿色半透明: 在凸包内的 cell (保留)
- 黄色半透明: 在 AABB 内但在凸包外的 cell (被排除)
- 标注: 每个 GT 的类别名和 cell 数量 (AABB 数 / 凸包数)
```
