## 任务清单

### 任务 1: AABB → 旋转多边形标签修复
**文件**: `GiT/mmdet/datasets/generate_occ_flow_labels.py`
**位置**: `_compute_valid_grid_ids` 方法 (约 L507)

**当前问题**: 使用 `min_u, max_u, min_v, max_v` (AABB) 分配 cell. 旋转车辆的 AABB 面积约为实际面积 2 倍, 导致大量假阳性标签, 系统性拖低 Precision.

**修复方案**:
1. 获取 3D box 投影到 BEV 的旋转多边形顶点
2. 使用 `shapely.geometry.Polygon` 或手动实现旋转矩形判断
3. 判断 cell 中心是否在旋转多边形内 (而非 AABB 内)
4. 保持向后兼容: 添加参数 `use_rotated_polygon=True` 控制新旧行为

**验证**: 对比修复前后标签数量变化, 预期标签减少 30-50% (尤其 truck, trailer).

### 任务 2: BUG-11 修复
**文件**: `GiT/mmdet/datasets/generate_occ_flow_labels.py`
**位置**: L77 (或附近, 类别顺序默认值)

**当前问题**: 默认 `classes=["car","bus","truck","trailer"]` 与 config 中 `["car","truck","bus","trailer"]` 顺序不一致. 如果新 config 忘记显式传入, truck 和 bus 标签会互换.

**修复方案**: 删除默认值, 强制显式传入. 如果未传入则 raise ValueError.
