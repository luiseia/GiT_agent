## 4. 对比分析

### 4.1 类竞争假说: REJECTED ❌

| | car_P | 对比 P5b@3000 |
|---|-------|---------------|
| P5b@3000 (10类) | 0.107 | baseline |
| Plan K (1类) | 0.063 | **-41%** |

单类 car 训练精度更低。原因分析:
- num_vocal 从 230→221, 所有 bin token 含义重映射
- 10→1 类的 vocab 变化破坏了 P5b@3000 checkpoint 学到的 token 分布
- **结论: 类竞争不是精度瓶颈, vocab 兼容性是更大问题**

### 4.2 投影层宽度假说: CONFIRMED ✅

| | car_P | 对比 P5b@3000 |
|---|-------|---------------|
| P5b@3000 (proj=1024) | 0.107 | baseline |
| Plan L@1000 (proj=2048) | **0.140** | **+31%** |
| Plan L@2000 (proj=2048) | **0.111** | **+4%** |

宽投影层从随机初始化仅 1000 iter 就超越了 P5b 基线:
- **4096→2048→768 保留了更多 DINOv3 信息**
- 即使最终@2000 (0.111), 仍优于 P5b 1024 版本 (0.107)
- **结论: 投影层是关键瓶颈, 建议 P7 采用 proj_hidden_dim=2048**

### 4.3 在线 vs 离线 DINOv3: 最终对比 (ORCH_016 COMPLETED)

| | car_P@500 | car_P@1000 | car_P@2000 | offset_th@2000 | bg_FA@2000 |
|---|-----------|------------|------------|----------------|------------|
| Plan K (预提取) | 0.064 | 0.047 | **0.063** | **0.191** | **0.166** |
| Plan M (在线 unfreeze) | 0.052 | 0.049 | 0.049 | 0.223 | 0.188 |
| Plan N (在线 frozen) | 0.051 | 0.050 | 0.045 | 0.217 | 0.198 |

**最终结论**:
1. **在线 DINOv3 可行但不优于预提取**: K(0.063) > M(0.049) > N(0.045)
2. **M vs N 无显著差异**: unfreeze 增益极微 (+0.004), 不值得额外开销
3. **预提取在 bg_FA 和 offset_th 上更优**: K 的 bg_FA=0.166 < M=0.188 < N=0.198
4. **在线模式额外开销**: +14 GB 显存, 2x 时间
5. **建议**: 继续使用预提取特征; 仅在需要数据增强时考虑在线模式 (BUG-16 场景)
