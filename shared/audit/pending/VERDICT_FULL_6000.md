# 审计判决 — FULL_6000

## 结论: CONDITIONAL

**条件**: 继续训练不中断，@8000 决策矩阵仍适用。bg_FA=0.331 是多类学习的预期代价，但若 @8000 仍 >0.30 则需干预 per_class_balance。

---

## Q1: bg_FA=0.331 是否告警？

**不告警。这是 (a) + (c) 的组合效应。**

### 因果链

```
多类爆发 (bus_R: 0→0.287, ped_R: 0.026→0.145, cone_R: 0→0.160)
→ 模型大量预测新类前景 (大部分是 FP)
→ 背景 cell 被误标为前景
→ bg_FA 从 0.199 → 0.331
```

bg_R + bg_FA = 1.0 (0.669 + 0.331 = 1.0)。bg_FA=0.331 意味着 33% 的背景 cell 被预测为前景。

### 量化验证

@4000 只有 car/truck 有实质预测 → bg_FA=0.199
@6000 新增 bus/ped/cone 的大量 FP → bg_FA=0.331

每个新类带来的 FP 增量:
- bus: P=0.009, R=0.287 → TP≈6052, FP≈6052/0.009-6052≈666K? 数字不合理，说明 bus 的 FP 量极大
- ped: P=0.024, R=0.145 → TP≈2340, FP≈2340/0.024-2340≈95K
- cone: P=0.001, R=0.160 → TP≈774, FP≈774K

这些新类的海量 FP 直接推高了 bg_FA。

### 是否是模型崩溃？

**不是。** 模型崩溃的标志是 car_P/R 同时暴跌。实际上:
- car_P: 0.078 → 0.090 (+15%) ← 改善
- car_R: 0.419 → 0.455 (+8.6%) ← 改善
- 模型在 car 上变得更好，同时在学习新类

### BUG-17 的角色

per_class_balance sqrt 鼓励模型过度预测稀有类。但 bg_FA 恶化的 **主要驱动力** 是多类学习本身 (即使没有 sqrt balance，新类 FP 也会推高 bg_FA)。sqrt balance 加剧了这个问题。

### 预判

bg_FA 在 LR decay @17000 后 **应当收敛**。理由:
1. LR 降低 10x 会稳定预测，减少 FP spam
2. 更多训练让模型学会区分 true/false foreground
3. Mini 阶段也经历过 bg_FA 恶化再收敛的模式

**@8000 bg_FA 判断阈值:**
- < 0.30: 正常收敛中
- 0.30-0.35: 可接受，继续到 LR decay
- \> 0.35: 需关注，考虑降低 bg_balance_weight 或调整 per_class_balance

## Q2: car_P=0.090 (+15%) 是否可信？

**可信度: 中等偏高。需 @8000 确认。**

### 评判规则应用

规则 #3: 单次 >15% 变化 "可能有意义，但需排除超参/BUG 原因"。

15.4% 的变化幅度刚好触发规则 #3。结合以下上下文:

| 支持可信的证据 | 反对可信的证据 |
|--------------|--------------|
| car_R 同向改善 (+8.6%) | 仅 500 optimizer steps 差距 |
| 连续 2 个 eval 持平后突破 | DDP BUG-33 不确定性 |
| 多类学习没有拖累 car | 单次 eval 点 |

**P 和 R 同时改善是最强信号**。如果是随机波动，P↑ 时通常 R↓ (precision-recall tradeoff)。P 和 R 同向 ↑ 表明模型的 car 检测能力在实质性改善。

### 预判

@8000 car_P 预测区间: **0.08-0.12**
- 如果 @8000 car_P ≥ 0.10 → 确认 @6000 的改善是趋势性的
- 如果 @8000 car_P 回落到 <0.08 → @6000 是波动

## Q3: Offset 回归恶化原因

**主因: 新类几何差异拉偏平均值。**

### 新类几何属性对比

| 类别 | 典型宽度 (m) | 典型长度 (m) | BEV 特征 |
|------|------------|-------------|---------|
| car | 1.8-2.0 | 4.0-5.0 | 标准尺寸，占训练多数 |
| bus | 2.5-3.0 | 10-12 | **极长** |
| pedestrian | 0.4-0.6 | 0.4-0.6 | **极小** |
| traffic_cone | 0.2-0.3 | 0.2-0.3 | **微小** |

offset 指标是 **跨类平均值**:
- @4000 主要由 car 贡献 (truck 少量) → offset 反映 car 精度
- @6000 新增 bus/ped/cone 的 FP → 这些 FP 的几何属性与 GT 差距极大 → 拉高平均 offset error

### 验证

off_cy 反而改善了 (0.097→0.082, -15%)。这看起来矛盾，但:
- cy (纵深) 是所有类别的共同难题，不受新类影响
- cx, w, h 受新类极端尺寸影响更大

**结论: offset 恶化不代表模型回归能力退化，是新类 FP 的统计效应。** 需要 per-class offset 数据才能确认 car 自身的 offset 是否退化。

## Q4: bicycle 振荡 (0→0.191→0)

**BUG-17 周期性振荡的直接证据。**

@4000: bicycle spam 阶段 → R=0.191, P=0.001, 154K FP
@6000: bicycle 抑制阶段 → R=0, P=0

振荡周期: ~2000 iter (约 500 optimizer steps)。与 mini 阶段 ~1000 iter 周期一致 (mini 无 accumulative_counts)。

### 机制

```
Phase A (@4000): bicycle GT 出现在 batch 中
→ sqrt balance: bicycle 获得 max weight (~11x car)
→ 模型大量预测 bicycle → 154K FP
→ bicycle loss 降低 (预测多 → count 高 → weight 低)

Phase B (@6000): 模型收缩 bicycle 预测
→ bicycle FP 消失
→ 但 bicycle GT 仍存在
→ 下一个周期 bicycle 将重新爆发
```

**预判: @8000 bicycle 可能重新出现。** 这种振荡会持续直到 LR decay 或 per_class_balance 被修改。
