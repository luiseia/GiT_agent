## Conductor 分析

### 宽投影 2048 的问题可能是什么？

1. **容量过大**: 2048 中间层给了小类更多子空间来"偷走" car 的梯度, 加剧类振荡。P5b 1024 的"压缩"反而迫使模型优先学好 car。
2. **去 GELU 的代价**: 无激活函数的纯线性变换可能丧失了非线性特征提取能力, 导致更多 FP → bg_FA 恶化。
3. **Mini 特异性**: 323 图上 2048 维度过大, 容易过拟合。Full nuScenes 可能完全不同。

### 建议

1. **不要急于否定 P6 2048**: offset 精度的巨大提升是真实的, 这在 BEV 任务中可能比 car_P 更重要
2. **P6 可以继续到 @6000**: 低成本保险, 观察第二次 LR decay 效果
3. **Full nuScenes 优先用 P5b config 做基线**: 然后再测试 2048
4. **Plan O 仍然需要**: 在线 vs 预提取的问题与 1024/2048 选择独立
5. **考虑 Plan P**: 2048 + GELU 的组合, 分离两个变量
